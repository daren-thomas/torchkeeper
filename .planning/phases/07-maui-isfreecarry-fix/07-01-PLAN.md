<objective>
Propagate `IsFreeCarry` to the MAUI-local model, DTO, and service layer so free-carry status round-trips correctly through save/load in the running app. Closes the GEAR-01 gap where manually-flagged items silently lose their flag on reload.

Purpose: The Core library was updated in Phase 6, but the MAUI project maintains parallel copies under the same namespaces. Due to CS0436 shadowing, the MAUI-local definitions win at compile time. Without this fix, `IsFreeCarry` is silently dropped on every save/load cycle.

Output: Four updated MAUI-local files with `IsFreeCarry` parity, verified by MAUI build and existing test suite.
</objective>
