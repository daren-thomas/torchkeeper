### Integration Finding: Test Stub Coverage Gap

**Finding:** `TestFileCommandVM.BuildCharacterFromViewModel()` and `TestFileCommandVM.LoadCharacter()` in `CharacterViewModelFileCommandTests.cs` omit the `Talents` field. The production implementations include it (line 234 and line 275 of `CharacterViewModel.cs` respectively).

**Consequence:** The 6 file-command unit tests would not catch a regression where `Talents` is silently dropped from the save or load path.

**Severity:** Medium — tech debt only. Production code is correct. The data-layer round-trip is covered by Phase 05's `CharacterFileServiceTests`. No runtime breakage.

**Affected requirements:** FILE-01, FILE-02, TLNT-01.

**Fix:** Add `Talents = Talents` to `TestFileCommandVM.BuildCharacterFromViewModel()` (after `Notes = Notes,` around line 138) and `LoadCharacter()` (after `SpellsKnown` assignment around line 169), plus one assertion in an existing save test.
