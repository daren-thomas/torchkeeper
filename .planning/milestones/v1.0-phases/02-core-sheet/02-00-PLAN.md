Create SdCharacterSheet.Tests/ViewModels/ directory (if it doesn't exist) and two test files:

**CharacterViewModelTests.cs** — contains a private `TestCharacterVM` inner class mirroring the CharacterViewModel computed properties (TotalSTR via bonus filtering, ModSTR via floor division, GearSlotTotal, CoinSlots, CurrentHP no-floor, LoadCharacter field propagation). Tests reference TestCharacterVM and will be converted to reference the real CharacterViewModel in plan 02-01 Task 1. All tests decorated with [Trait("Category", "Unit")].

Key computed property implementations for TestCharacterVM (must match what plan 02-01 will implement — these are the contracts):
- `TotalSTR`: BaseSTR + sum of bonus values where BonusTo starts with "STR:"
- `ModSTR`: `(int)Math.Floor((TotalSTR - 10.0) / 2.0)` — uses Math.Floor to correctly handle odd values below 10
- `GearSlotTotal`: `Math.Max(TotalSTR, 10)` — STR score or 10, whichever is higher
- `CoinSlots`: `Math.Max(GP - 100, 0) / 100 + Math.Max(SP - 100, 0) / 100 + Math.Max(CP - 100, 0) / 100` — integer division floors automatically
- `GearSlotsUsed`: `GearItems.Sum(g => g.Slots) + CoinSlots`
- Bonus value parsing: parse the numeric part of BonusTo (e.g. "STR:+2" → +2, "STR:-1" → -1) using `int.Parse(b.BonusTo.Split(':')[1])`
- `LoadCharacter`: sets all identity/stat/HP/currency fields and rebuilds GearItems from Character.Gear and Character.MagicItems

For the NegativeHP test: create a VM instance, set CurrentHP = -5, assert CurrentHP == -5.
For the LoadCharacter test: create VM, call LoadCharacter with a fully-populated Character, assert Name/Class/Level/BaseSTR etc. match.

**GearItemViewModelTests.cs** — contains a private `TestGearItemVM` inner class that wraps either a GearItem or MagicItem. Tests assert field mapping for both source types, with ItemType defaulting to "" for MagicItem. All tests decorated with [Trait("Category", "Unit")].
