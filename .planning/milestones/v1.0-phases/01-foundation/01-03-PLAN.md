  <implementation>
    Replace the empty CharacterFileService stub with a real implementation.

    Class design:
    - Private static readonly JsonSerializerOptions SaveOptions: WriteIndented=true,
      DefaultIgnoreCondition=JsonIgnoreCondition.WhenWritingDefault.
    - Private static readonly JsonSerializerOptions LoadOptions: PropertyNameCaseInsensitive=true,
      AllowTrailingCommas=true.
    - Public CharacterSaveData MapToDto(Character character) — maps domain model to DTO.
      Map each field by name. Nested lists: map BonusSource→BonusSourceData, GearItem→GearItemData,
      MagicItem→MagicItemData field-by-field. Return new CharacterSaveData { Version=1, ... }.
    - Public Character MapFromDto(CharacterSaveData dto) — maps DTO back to domain model.
      Reverse of MapToDto. Return new Character { ... }.
    - Public async Task SaveToStreamAsync(Character character, Stream stream) — converts Character to DTO,
      serializes to JSON, writes UTF-8 bytes to the provided stream. Use JsonSerializer.SerializeAsync.
    - Public async Task<CharacterSaveData?> LoadFromStreamAsync(Stream stream) — deserializes stream
      to CharacterSaveData using LoadOptions. Returns null if stream is empty or parse fails.

    Do NOT put FilePicker or FileSaver calls in CharacterFileService — those are added in Plan 04.
    The streaming API is the testable core; Plan 04 wraps it with native dialog calls.

    Do NOT use JsonSerializerOptions.IgnoreNullValues (deprecated in .NET 10 — use DefaultIgnoreCondition).
    Do NOT create new JsonSerializerOptions per call (use the static readonly instances).
