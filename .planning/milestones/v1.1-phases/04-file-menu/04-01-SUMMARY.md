## Decisions Made

- MauiCharacterFileService is resolved via cast `(MauiCharacterFileService)sp.GetRequiredService<CharacterFileService>()` because DI registration uses base type as key but the constructor needs the concrete type for `OpenAsync()`
- `BuildCharacterFromViewModel()` builds fresh Character from observable properties — not the `Character` backing field which is stale after edits
- GearItemSource enum is the canonical discriminator for splitting gear vs magic items on save
- Save/Open/Import use `Order="Secondary"` (overflow menu) on mobile to avoid crowding the primary toolbar
