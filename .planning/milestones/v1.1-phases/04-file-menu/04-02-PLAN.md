    Plan 01 wired three file commands into CharacterViewModel and AppShell:
    - SaveCommand: builds Character from VM state, calls CharacterFileService.SaveAsync (native save dialog)
    - LoadCommand: calls MauiCharacterFileService.OpenAsync (native file picker), calls LoadCharacter
    - ImportCommand: calls MauiImportFileService.ImportAsync (JSON file picker + ShadowdarklingsImportService), calls LoadCharacter
    - AppShell has Save/Open/Import in File menu (desktop) and toolbar overflow (mobile)
  </what-built>

  <how-to-verify>
    Run the app on macOS (or your preferred platform):

    1. FILE-01 (Save):
       a. Edit the character: set Name to "Test Save", Level to 5, add a gear item
       b. File menu -> Save... (or toolbar overflow -> Save on mobile)
       c. A native save dialog should appear with suggested filename "Test Save.sdchar"
       d. Save the file to a known location (e.g., Desktop)
       e. Verify the .sdchar file exists on disk and is valid JSON (open in text editor)

    2. FILE-02 (Open/Load):
       a. Change the character name to "Changed After Save"
       b. File menu -> Open... (or toolbar overflow -> Open on mobile)
       c. A native file picker should appear filtered to .sdchar files
       d. Select the file saved in step 1
       e. Verify the sheet now shows Name="Test Save", Level=5, and the gear item from step 1

    3. FILE-03 (Import):
       a. File menu -> Import from Shadowdarklings... (or toolbar overflow -> Import on mobile)
       b. A native file picker should appear filtered to .json files
       c. Select a Shadowdarklings JSON export file (use examples/Brim.json if available, or TorchKeeper.Tests/TestData/Brim.json)
       d. Verify the sheet populates with the imported character data (name, stats, gear, etc.)

    4. Cancellation:
       a. File menu -> Save... then click Cancel in the save dialog
       b. Verify no crash and the current character is unchanged
       c. File menu -> Open... then click Cancel in the file picker
       d. Verify no crash and the current character is unchanged

    5. Menu visibility:
       a. On macOS/Windows: verify "Save...", "Open...", "Import from Shadowdarklings..." appear in the File menu bar, with a separator before "Export As..."
       b. On iOS/Android (if available): verify Save, Open, Import appear in the toolbar overflow (three-dot) menu
